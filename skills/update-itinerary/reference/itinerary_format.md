# 總行程格式規範（itinerary.md）

`itinerary.md` 是行程索引，不放細節，每列連結到對應的每日檔案。

---

## 範本

```markdown
# {地區描述} {N}天{N}晚行程

## 行程概覽

- **日期：** {YYYY/MM/DD}–{YYYY/MM/DD}（{N} 天 {N} 晚）
- **住宿安排：** {城市} {N}晚 → {城市} {N}晚 → ...
- **去程：** {航班號}｜{MM/DD} {HH:MM} {出發城市} → {HH:MM} {抵達城市}
- **回程：** {航班號}｜{MM/DD} {HH:MM} {出發城市} → {HH:MM} {抵達城市}

## 行程總覽

| 日期 | 住宿 | 行程 |
|---|---|---|
| [{MM/DD}](days/{MMDD}.md) | {城市} | {一行摘要（50字以內）} |

## 行程原則

- {原則}
```

---

## 規則

- 標頭資訊放在 `## 行程概覽` 區塊，以粗體 bullet list 呈現
- 標題列日期使用 Markdown 連結，指向 `days/` 資料夾的對應檔案
- 行程欄位只寫一行摘要，細節放在每日檔案
- 住宿欄位：回程日填 `—`
- 去回程資訊若有多段（含轉機），每段各一行
- 天數與晚數依實際航班抵達／離開日計算，抵達當天算第1天
