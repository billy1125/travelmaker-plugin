# 交通指南格式規範（transportations/）

交通指南記錄**查證過的路段交通資訊**，一個路段一個檔案，隨討論逐段累積。查證過一次就歸檔，之後的討論直接取用，不必重查。

---

## 檔案命名

`{起點}_{訖點}.md`

- 起訖點使用機場 IATA 代碼或地點英文名（駝峰式），以底線連接
- 同一路段雙向共用一個檔案，不分去回程

範例：`OKJ_OkayamaStation.md`（岡山桃太郎機場 ↔ 岡山站）

---

## 範本

```markdown
# {起點} ↔ {訖點} {交通方式}指南

## 路線概覽

| 項目 | 內容 |
|------|------|
| 路線 | {起點（站牌/月台）} ↔ {訖點（站牌/月台）} |
| 所需時間 | 約 {N} 分鐘 |
| 票價 | 大人 {N} 円・兒童 {N} 円 |
| 預約 | {需 / 不需預約} |
| 營運公司 | {公司名} |
| 班次頻率 | 約每 {N} 分鐘一班 |

> ⚠️ 票價與班次以出發前官方公告為準。

## 站牌／乘車位置

- **{起點}**：{具體位置描述}（附 Google Maps 搜尋連結）
- **{訖點}**：{具體位置描述}

## 付款方式

{IC 卡、售票機、現金等，表格或 bullet list}

## 時刻表

> ⚠️ 註明時刻表版本日期；若旅行期間的版本尚未公告，提醒出發前再確認。

{去回方向各一張表，只列與行程相關的班次密度即可}

## 本次行程使用規劃

{對應到具體的行程日（Day N、MM/DD），列出當天搭乘的時間流程表，
含風險提醒（如班次空窗、轉乘緩衝不足時的替代方案）}

## 官方資訊來源

{官方網站、時刻表頁面連結，每行一個}
```

---

## 規則

- `## 路線概覽` 與 `## 官方資訊來源` 為必填，其餘區塊視路段複雜度加入
- 票價、班次、所需時間等數字必須有來源依據，並加註「以出發前官方公告為準」
- `## 本次行程使用規劃` 將通用交通資訊對應到本次行程的具體日期，是檔案最有價值的部分，有相關行程日就要寫
- 時刻表若非旅行期間的版本，必須明確標註版本日期與「出發前確認」提醒
- 查證來源的網站應同步經 `/save-website-abstract` 歸檔摘要
- 地點附 Google Maps 搜尋連結（格式同 `day_format.md`）
- 所有文字使用繁體中文
